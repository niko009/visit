# 🎨 TinaCMS - Руководство по использованию

## 🚀 Доступ к админ-панели

### Локальная разработка:
1. Запустите dev-сервер: `npm run tina`
2. Откройте в браузере: **http://localhost:4321/admin**
3. Админ-панель автоматически загрузится

### Прямые ссылки:
- **Админка**: http://localhost:4321/admin/index.html
- **GraphQL Playground**: http://localhost:4321/admin/index.html#/graphql
- **API**: http://localhost:4001/graphql

---

## 📝 Редактирование контента

### Услуги (Services)
1. Перейдите в раздел **"Услуги"** в админ-панели
2. Нажмите **"Create New"** или выберите существующую услугу
3. Заполните поля:
   - **Название услуги** (обязательно, 3-100 символов)
   - **Цена** (в рублях, целое число)
   - **Длительность** (в минутах)
   - **Описание** (20-1000 символов)
   - **Изображение** (загрузите картинку JPG/PNG)
   - **Порядок отображения** (опционально, для сортировки)
   - **Активна** (галочка - показывать на сайте)
4. Нажмите **"Save"**

**Результат:** JSON-файл будет создан в `src/content/services/`

---

### Мастера (Masters)
1. Перейдите в раздел **"Мастера"**
2. Нажмите **"Create New"** или выберите существующего
3. Заполните поля:
   - **Полное имя** (обязательно)
   - **Должность** (например: "Мастер перманентного макияжа")
   - **Опыт** (количество лет)
   - **Биография** (подробное описание, 50-2000 символов)
   - **Фотография** (портретное фото)
   - **Instagram / Telegram** (опционально, полные ссылки)
   - **Порядок отображения**
   - **Активен** (показывать на сайте)
4. Нажмите **"Save"**

**Результат:** JSON-файл будет создан в `src/content/masters/`

---

## 📂 Загрузка изображений

### Через TinaCMS:
1. В поле изображения нажмите **"Upload"**
2. Выберите файл с компьютера
3. Изображение автоматически сохранится в `public/uploads/`

### Напрямую:
Положите изображения в:
- `src/content/services/[имя-файла].jpg` — для услуг
- `src/content/masters/[имя-файла].jpg` — для мастеров

В JSON-файле укажите путь: `"image": "./имя-файла.jpg"`

## 🔄 Как работает Git-based workflow

### Автоматическое сохранение:
1. Вы редактируете контент в TinaCMS
2. При нажатии **"Save"** TinaCMS:
   - ✅ Создает/обновляет JSON-файл
   - ✅ Делает Git commit с описанием изменений
   - ✅ Валидирует данные по Zod-схеме
3. Astro автоматически перезагружает страницу (hot reload)

### История изменений:
```bash
# Посмотреть последние изменения
git log --oneline

# Откатить изменение
git revert <commit-hash>

# Посмотреть diff
git diff
```

## ⚙️ Настройка схем

### Изменение полей коллекции:
Отредактируйте `tina/config.ts`:

```typescript
{
  type: 'string',
  label: 'Новое поле',
  name: 'newField',
  required: true,
}
```

**После изменений:**
1. Перезапустите `npm run tina`
2. Обновите Zod-схему в `src/content/config.ts`

## 🔒 Безопасность

### Локальная разработка:
- TinaCMS работает без аутентификации
- Все изменения сохраняются локально в Git
- Доступ только с вашего компьютера

### Production (GitHub):
Для работы на продакшене нужно:
1. Создать GitHub OAuth App
2. Получить `TINA_CLIENT_ID` и `TINA_TOKEN`
3. Добавить в `.env`:
   ```
   TINA_CLIENT_ID=your-client-id
   TINA_TOKEN=your-token
   ```

## 📊 Валидация данных

### Автоматическая проверка:
- ✅ Обязательные поля не могут быть пустыми
- ✅ Цены и длительность — только положительные числа
- ✅ URL соцсетей проверяются на корректность
- ✅ Длина текстовых полей ограничена

### При ошибке:
TinaCMS покажет сообщение с описанием проблемы. Исправьте и сохраните снова.

## 🆘 Решение проблем

### TinaCMS не загружается:
```bash
# Проверьте, запущен ли dev-сервер
npm run tina

# Очистите кеш
rm -rf node_modules/.vite
npm run dev
```

### Изображения не отображаются:
- Проверьте путь в JSON: `"image": "./filename.jpg"`
- Убедитесь, что файл существует рядом с JSON

### Git конфликты:
```bash
# Посмотреть статус
git status

# Откатить изменения
git reset --hard HEAD
```
